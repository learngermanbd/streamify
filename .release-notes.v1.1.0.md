## What's new — v1.1.0

### Android 16 (API 36) platform migration

This release moves the app to Android 16 ahead of the **August 31, 2026** Play Store deadline for `targetSdk = 36`.

**Build config**
- `compileSdk` 35 → **36**
- `targetSdk` 35 → **36**
- `versionCode` 1 → 2, `versionName` 1.0.0 → 1.1.0
- Gradle 8.11.1 → **8.13**
- AGP 8.7.3 → **8.9.2**
- NDK r27c retained (AGP 8.5+ injects 16 KB page-size linker flag automatically)

**Library updates**
- Media3 1.5.1 → **1.6.0**
- Room 2.6.1 → **2.7.0**
- Firebase BoM 33.7.0 → **34.0.0**
- Material 1.12.0 → **1.13.0** (provides `enableEdgeToEdge()` opt-in helper)
- Navigation 2.8.4 → **2.9.0** (predicate-back insets fix)
- Lifecycle 2.8.7 → **2.9.0**
- DataStore 1.1.1 → **1.1.4**, Gson 2.10.1 → **2.11.0**
- New: `androidx.activity:activity-ktx` 1.9.3 (for `enableEdgeToEdge()`)
- New: `androidx.fragment:fragment-ktx` 1.8.5 (pin to match Navigation 2.9 floor)

**Now enforced on Android 16**
- **Predictive Back** — `OnBackPressedDispatcher` auto-bridges. Explicit `enableOnBackInvokedCallback="false"` on `UpdateActivity` / `CrashActivity` so forced updates and crash reports don't animate the predictive gesture.
- **Edge-to-edge** — `MainActivity` and `PlayerActivity` swap `WindowCompat.setDecorFitsSystemWindows` for `enableEdgeToEdge()`. Existing `WindowInsetsCompat` listeners on top / bottom bars absorb the new system-bar insets.
- **16 KB page alignment** — AGP-injected linker flag covers our arm64 native libs.

**Cleanup**
- Removed `androidx.security:security-crypto` (1.1.0-alpha06, no production callers; `MasterKey.Builder` + `EncryptedSharedPreferences` can throw `KeyStoreException` on Android 16's stricter keystore key-commitment rules).
- `TokenManager` now uses plain `SharedPreferences("streamify_auth", MODE_PRIVATE)`; the dead encrypted-prefs wrapper is gone.
- ProGuard keep-rule typo (`ui.crash.CrashActivity`) fixed.
- `StreamifyApp.onCreate` now installs the `CrashHandler` before any other init step + per-step `runStep` guards so init faults surface as a Sentry event instead of a silent process death.
- `SplashActivity` surfaces init errors via the existing error card + Retry button.

### Upgrading from v1.0.0

Auth-token storage backend was migrated in this release. Users upgrading from an earlier install will see a **forced sign-in on first launch of v1.1.0** — this is expected. Server-side token revocation remains the strong defense.
